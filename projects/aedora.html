üìñ Project Overview

Aedora is an interactive AI platform that transforms static PDF books into dynamic conversations. Users can upload any book (PDF), automatically extract character personas, and chat with them in real-time.

Unlike generic chatbots, Aedora uses a Retrieval-Augmented Generation (RAG) pipeline to ensure characters remain strictly "in-character" and grounded in the text, significantly reducing hallucinations while maintaining a natural conversational flow.

üöÄ Key Features

  • PDF-to-Persona Engine: Automatically ingests raw PDF text, identifies main characters, and generates detailed JSON profiles (traits, relationships, voice)
  • Grounded RAG Chat: Answers are constrained to retrieved book passages, ensuring the character knows only what has happened in the story
  • Persistent Memory: Saves chat history and character profiles using persistent vector storage and SQL, eliminating the need to re-process books
  • Smart Fallbacks: Characters gracefully handle out-of-scope questions (e.g., "I don't recall that happening in my story") rather than hallucinating facts
  • Streaming Responses: Real-time token streaming for a responsive user experience

üß† System Architecture & Workflow

The core of Aedora is a multi-stage pipeline designed for accuracy and speed:

1. Ingestion & Chunking

  • Books are uploaded and text is extracted using a dual-extraction approach
  • Text is split into semantic chunks safe for embedding context windows

2. Vector Indexing

  • Chunks are embedded using text-embedding-3-small and stored in ChromaDB
  • Batch processing is used to handle large texts efficiently without hitting API rate limits

3. Character Profiling (The "Secret Sauce")

  • Before chatting, the system analyzes the text to build structured Persona Profiles
  • System Prompt Strategy: These profiles are fed into the chat system prompt to enforce tone, voice, and narrative constraints

4. Retrieval-Augmented Generation (RAG)

User Query ‚Üí Vector Search (Retrieve top relevant book passages) ‚Üí LLM Generation (Context + Persona + Query) ‚Üí In-Character Response

üí° Challenges & Optimizations

  • Hallucination Control: Implemented a strict "Persona-Driven" system prompt that forces the model to use only retrieved context
  • Optimizing Throughput: Used asynchronous processing and batching for embedding generation to prevent timeouts on large book uploads
  • Caching Strategy: Implemented a caching layer for character profiles. Once a book is processed, subsequent chats load instantly without re-running expensive extraction calls
  • PDF Complexity: Dual-layer extraction strategy (PyPDF2 + pdfplumber) handles complex PDF layouts and improves data ingestion reliability

üéØ Results & Impact

Aedora successfully demonstrates the power of RAG-based AI applications in creating engaging, contextually accurate conversational experiences. The system's architecture ensures:

  • Minimal hallucinations through strict context grounding
  • Fast response times via intelligent caching and batching
  • Scalable design that can easily transition from local SQLite to cloud PostgreSQL
  • Cost-effective operation through optimized embedding and LLM usage
